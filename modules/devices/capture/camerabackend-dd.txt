= CameraBackend

== Description

I am a camera backend interface.
I create `CameraStreamBackend` instances for camera device ids and provide human-readable device descriptions.

I collaborate with `MediaCapture`, which uses me to create and manage per-device streams.
I collaborate with concrete backend implementations such as `QtCameraBackend` and `NullCameraBackend`.
I do not manage capture lifetime; `MediaCapture` controls when streams start and stop.

State model:
- Thread affinity: UI thread.
- Lifetime/ownership: owned by `MediaCapture`.
- Re-entrancy: not re-entrant.

== Public interface

- `QString descriptionForId( const QString &deviceId ) const` returns a description or an empty string.
- `std::unique_ptr<CameraStreamBackend> createStream( const QString &deviceId, QObject *parent ) const` returns a stream or null.

Errors:
- Returning null from `createStream` indicates the device id is not available.

== Requirements

- Language: C++17.
- Qt: 6.x.
