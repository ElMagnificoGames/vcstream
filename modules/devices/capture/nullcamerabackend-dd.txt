= NullCameraBackend

== Description

I am a null camera backend.
I provide a deterministic, synthetic video stream that behaves like a camera device without requiring hardware.

I collaborate with `MediaCapture` which uses me as one possible camera backend implementation.
I do not enumerate physical devices, and I do not attempt to emulate a specific camera driver.

State model:
- Thread affinity: UI thread.
- Lifetime/ownership: owned by the caller that selects the camera backend for `MediaCapture`.
- Re-entrancy: not re-entrant.

== Public interface

- `QString descriptionForId( const QString &deviceId ) const` returns a human-readable description.
- `std::unique_ptr<CameraStreamBackend> createStream( const QString &deviceId, QObject *parent ) const` creates a synthetic stream.

Errors:
- This backend does not fail based on `deviceId`; it always produces a stream.

== Implementation details

- Uses a timer-driven frame generator to emit `QVideoFrame` instances.
- Frames change colour over time so consumers can detect liveness.

== Requirements

- Language: C++17.
- Qt: 6.x.
