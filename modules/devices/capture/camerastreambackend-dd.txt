= CameraStreamBackend

== Description

I am a camera stream backend interface.
I represent a single running camera stream that can be started and stopped and that emits decoded video frames.

I collaborate with `MediaCapture`, which owns per-device stream instances and forwards frames to multiple view sinks.
I collaborate with backend implementations such as `QtCameraBackend` and `NullCameraBackend`.
I do not select devices, I do not enumerate devices, and I do not render video.

State model:
- Thread affinity: UI thread.
- Lifetime/ownership: owned by the per-device stream object inside `MediaCapture`.
- Re-entrancy: not re-entrant.

== Public interface

- `void start()` begins producing frames.
- `void stop()` stops producing frames.
- `videoFrameChanged( const QVideoFrame &frame )` is emitted when a new frame is available.
- `QString errorText` is a user-facing message for capture liveness or format failures.
- `errorTextChanged()` is emitted when `errorText` changes.

Thread safety:
- Thread-safe: No.
- Notes: Intended for UI-thread use. Implementations may receive frames on backend threads but should forward signals to the owning thread.

== Implementation details

- Backends may be hardware-backed (Qt Multimedia) or synthetic (`NullCameraBackend`).
- `MediaCapture` may attach multiple consumers to the same stream; the backend is not aware of consumers.

== Requirements

- Language: C++17.
- Qt: 6.x (QtCore, QtMultimedia).
