 = FontPreviewSafetyChecker

 == Description

 I am an abstract interface for running font preview safety checks.
 I provide a small asynchronous contract so other code can request a check for a font family and receive a completion signal.

 I collaborate with `FontPreviewSafetyCache`, which owns the policy and state machine for caching results and applying timeouts/backpressure.
 I collaborate with concrete implementations such as `FontPreviewSafetyCheckerDefault`.
 I do not define the check algorithm.
 I do not define caching or timeout policy.

 State model:
 - Thread affinity: calls to `startCheck` occur on the UI thread.
 - Lifetime/ownership: owned by a cache/supervisor (QObject parented) and lives for at least as long as any in-flight checks.
 - Re-entrancy: `startCheck` is not required to be re-entrant.
 - Concurrency: implementation-defined.

 == Public interface

 Functions:
 - `void startCheck( const QString &family, int pixelSize )`
   - Starts an asynchronous check for the given font family at the given pixel size.
   - Implementations MUST eventually emit `checkFinished( family, pixelSize, ok )` if the check completes.
   - Implementations MAY hang or never finish if the underlying font stack hangs; callers must not assume completion.

 Signals:
 - `checkFinished( const QString &family, int pixelSize, bool ok )`
   - Emitted when a check completes.
   - `ok=true` means "safe for preview" under the implementation's algorithm.
   - `ok=false` means "appears unsafe for preview" under the implementation's algorithm.

 Thread safety:
 - Thread-safe: No.
 - Calls MUST be made from the UI thread.

 Errors:
 - Implementations do not report hard errors via this interface.
 - Callers should use timeouts/backpressure at a higher level (`FontPreviewSafetyCache`).

 == Requirements

 - Language: C++17.
 - Qt: 6.x (QtCore).
