= AccentImageProvider

== Description

I am a QML image provider for accent colour picker UI.
I render a hue bar and an OKLCH chroma/lightness plane into `QImage` results, so QML can display a 2D picker without requiring GPU shaders.

I collaborate with theme preference UI and with OKLCH conversion utilities.
I do not persist settings, I do not handle pointer interaction, and I do not apply palettes.

State model:
- Thread affinity: UI thread (called by QML image loading).
- Lifetime/ownership: owned by a `QQmlApplicationEngine` via `addImageProvider()`.
- Re-entrancy: not re-entrant.

== Public interface

Registration:
- Register this provider with a `QQmlApplicationEngine` via `addImageProvider( "<name>", new AccentImageProvider() )`.

Functions:
- `QImage requestImage( const QString &id, QSize *size, const QSize &requestedSize )` returns an image for the requested id.

  Arguments:
  - `id` is the image id string (see Image ids).
  - `size` is an output pointer; if non-null it is filled with the returned image size.
  - `requestedSize` is the desired size.

Image ids:
- `hue` renders a horizontal hue gradient bar.
- `plane/<hueDegrees>` renders a chroma/lightness plane at the given hue.

Arguments:
- `hueDegrees` is parsed as a `double`.
  - If parsing fails, the provider uses 0 degrees.

Sizing:
- The provider uses `requestedSize`.
  - If `requestedSize.width()` or `requestedSize.height()` is <= 0, a conservative default is used.

Returns:
- Images are returned as `QImage` values.
  - For known ids, pixels are fully opaque (alpha 255).
  - For unknown ids, a transparent image is returned.

Thread safety:
- Thread-safe: No.
- Re-entrant: No.

Errors:
- Unknown ids return a transparent image.

== Implementation details

- Plane is rendered in OKLCH and converted to sRGB; out-of-gamut colours are fitted by reducing chroma.
- Rendering uses `QImage::Format_ARGB32_Premultiplied` for predictable results across platforms.

Picker coordinate contract:
- The plane renders:
  - x axis: chroma in [0, 0.37]
  - y axis: lightness in [0.08, 0.95], with y=0 at 0.95 (top) and y=height-1 at 0.08 (bottom)
- The hue bar renders a full 0..360 sweep at fixed lightness/chroma suitable for preview.

== Requirements

- Language: C++17.
- Qt: 6.x (QtGui, QtQml, QtQuick).
